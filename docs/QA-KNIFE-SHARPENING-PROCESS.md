# 칼 연마 프로세스 QA 테스트

## 📋 개요

칼가는곳 서비스의 핵심 프로세스인 칼 연마 신청부터 완료까지의 전체 플로우에 대한 QA 테스트 케이스입니다.

- **테스트 범위**: 홈 → 칼갈이 신청 → 로그인/회원가입 → 주소등록 → 결제
- **총 테스트 케이스**: 38개
- **통과**: 11개 (28.9%)
- **실패**: 27개 (71.1%)

---

## TC0001: 첫 칼갈이 신청하기 버튼 클릭

### 화면
[홈] → [칼갈이 신청]

### Action
첫 칼갈이 신청하기 버튼 클릭

### Expected Result
- [칼갈이 신청] 페이지로 전환
- 예약 섹션에 신청을 요청한 날짜와 근접한 시간으로 자동 반영

### 비고
예약 섹션에 신청을 요청한 날짜와 근접한 시간으로 자동 반영하도록 수정 필요

### 통과 여부
❌ **FALSE**

### 코멘트
현재 자동 시간 설정이 되지 않음

## TC0002: 예약 날짜 선택

### 화면
[칼갈이 신청]

### Action
예약 날짜 드롭다운 클릭 및 날짜 선택

### Expected Result
- 드롭다운 클릭 시 기존 시간 선택/칼 선택 섹션이 확장된 드롭다운 아래로 이동
- 한글 달력이 표시되며, 클릭한 날짜 당일이 임의로 선택됨
- 선택한 날짜 이전의 날짜는 선택 불가
- 날짜 선택 버튼을 클릭해야 드롭다운이 회수됨

### 비고
- 선택한 날짜 이전의 날짜를 선택할 수 없도록 수정 필요
- 날짜를 클릭 후 날짜 선택 버튼을 클릭해야 드롭다운이 회수되도록 수정 필요

### 통과 여부
❌ **FALSE**

### 코멘트
날짜 선택 UX 개선 필요

## TC0004: 칼 추가하기 드롭다운 클릭

### 화면
[칼갈이 신청]

### Action
칼 추가하기 드롭다운 클릭

### Expected Result
- 칼 추가하기 드롭다운 클릭 시 칼 추가 옵션이 아래에 확장
- 칼을 선택하지 않았을 시 비활성화된 바로 신청 버튼이 아래 위치
- 칼을 한 자루라도 선택 시(날짜와 시간 선택 후) 활성화된 바로 신청 버튼 위치

### 비고
- 칼을 선택하지 않았을 시 비활성화된 바로 신청 버튼이 아래에 위치하도록 수정 필요
- 칼을 한자루라도 선택 시(날짜와 시간 선택 후) 활성화된 바로 신청 버튼이 아래에 위치하도록 수정 필요

### 통과 여부
❌ **FALSE**

### 코멘트
- 현재 버전에선 드롭다운 형식이 아닌 바텀싯으로 올라옴
- 추후에 칼 추가하기 드롭다운이 아닌 칼 추가하기 버튼으로 전환 시 UX 측면에서 고려 가능
- 현 단계에선 드롭다운 방식으로 유지 (드롭다운에서 칼 종류를 선택 후 칼 수량을 추가하는 방식)
- "바로 신청하기" → "바로 신청"으로 수정 필요

## TC0005: 칼 종류 선택

- 총 금액 아래 "부가세 별도" 문구

### 비고
바텀싯에서 드롭다운으로 수정 필요

### 통과 여부
❌ **FALSE**

### 코멘트
바텀싯 방식 유지 결정
