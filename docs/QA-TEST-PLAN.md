## 🎯 우선순위별 수정 사항

### 🔴 Critical (긴급 - 서비스 불가)
1. **TC0061**: 쿠폰 사용해도 차감 안됨, 결제 적용 안됨
2. **TC0038**: 결제 후 페이지 이동 문제 (뒤로가기 불가)
3. **TC0065**: 휴대폰 번호 변경 시 인증 절차 없음
4. **TC0024**: 제한시간 지난 후 약관 확인 버튼 반응 없음

### 🟠 High (중요 - 사용자 경험 저해)
5. **TC0048**: 전체 읽음 버튼 작동 안함
6. **TC0053**: 연도 선택 드롭다운 클릭 안됨
7. **TC0056**: 같은 날짜에 서비스 2회 이상 시 날짜 중복 스택
8. **TC0063**: 기존 회원정보 확인 불가
9. **TC0064**: 이름 입력 시 확인 버튼 활성화 안됨

### 🟡 Medium (개선 필요)
10. **TC0002**: 날짜 선택 후 자동으로 드롭다운 회수 필요
11. **TC0003**: 시간 선택 시 자동으로 정오 12:00 선택 필요
12. **TC0040**: 가격 표시 형식 수정 필요 ([정상 4,000원] → 4,000원)
13. **TC0041**: 칼 상태 및 종류 관련 주의사항 오타 검토

## ✅ 테스트 진행 가이드

### 1. 테스트 시작 전 준비
```bash
# 개발 서버 실행
npm run dev

# 데이터베이스 마이그레이션 확인
npx supabase db push

# 테스트 데이터 확인
```

### 2. 테스트 수행 순서
1. **로그인/회원가입** 테스트 (TC0001 ~ TC0026)

3. **결제 프로세스** 테스트 (TC0025 ~ TC0038)
4. **이용내역** 테스트 (TC0050 ~ TC0057)
5. **알림** 테스트 (TC0046 ~ TC0049)
6. **내정보** 테스트 (TC0057 ~ TC0082)

### 3. 버그 발견 시
1. TC ID 기록
2. 재현 단계 상세 작성
3. 스크린샷/비디오 첨부
4. 우선순위 지정 (Critical/High/Medium/Low)
